# Автоматизация тестирования сервиса Кинопоиск (UI + API)

Этот проект является финальной работой по курсу автоматизации тестирования. В нем реализованы автоматизированные сценарии для проверки функциональности поиска, просмотра и работы с данными через интерфейс (UI) и программный интерфейс (API).

## 📌 Задачи проекта
- Реализовать от 5 UI-тестов на основе функционального чек-листа.
- Реализовать от 5 API-тестов на основе коллекции запросов.
- Обеспечить запуск тестов в разных режимах (UI, API, All).
- Соблюсти стандарты PEP8 и организовать отчетность в Allure.

## 🛠 Технологии
- **Язык**: Python 3.10+
- **Тестовый фреймворк**: Pytest
- **UI автоматизация**: Selenium WebDriver + Webdriver-manager
- **API автоматизация**: Requests
- **Отчетность**: Allure Framework
- **Стиль кода**: Flake8 (PEP8)

## 📁 Структура проекта
- `data/config.py` — конфигурационные данные (URL, токены, логины).
- `test/test_ui.py` — набор тестов для пользовательского интерфейса.
- `test/test_api.py` — набор тестов для проверки API эндпоинтов.
- `conftest.py` — настройки фикстур (инициализация и закрытие браузера).
- `pytest.ini` — конфигурация маркеров запуска.

## 🚀 Запуск тестов

Перед запуском установите зависимости:
```
pip install -r requirements.txt
```

### Режимы запуска:
1. **Запуск только UI-тестов**:
   ```
   pytest -m ui
   ```
2. **Запуск только API-тестов**:
   ```
   pytest -m api
   ```
3. **Запуск всех тестов сразу**:
   ```
   pytest
   ```

### Генерация отчета Allure:
```
pytest --alluredir=allure-results
allure serve allure-results
```
